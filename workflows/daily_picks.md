# Workflow: Daily Picks

**Run by:** Anyone on the team (usually Friend B who owns the output layer)
**When:** Each day there are NHL games, before puck drop
**Purpose:** End-to-end runbook from raw data → ranked picks → Google Sheet

---

## Pre-Flight Checklist

Before running, confirm:
- [ ] `.env` has `ODDS_API_KEY` and `GOOGLE_SHEETS_ID` set
- [ ] Virtual environment is active (`source venv/bin/activate`)
- [ ] `credentials.json` is in the project root (for Google Sheets)
- [ ] There are actually NHL games today (check NHL.com or the schedule step will exit early)

---

## Full Daily Run

Run these commands in order. Steps 2–4 can be parallelized in separate terminals.

```bash
# Step 1: Check if there are games today
python tools/fetch_nhl_schedule.py
```
If "No games found today" — stop here. No other steps needed.

```bash
# Step 2a: Pull player game logs (takes 3–5 minutes)
python tools/fetch_player_game_logs.py

# Step 2b: Pull team defense stats (run simultaneously with 2a)
python tools/fetch_team_defense_stats.py

# Step 2c: Pull MoneyPuck advanced stats (run simultaneously with 2a)
python tools/fetch_advanced_stats.py

# Step 3: Pull sportsbook odds (run after schedule is confirmed)
python tools/fetch_odds.py
```

Wait for all of the above to complete, then:

```bash
# Step 4: Build features
python tools/build_features.py

# Step 5: Generate predictions
python tools/predict_shots.py

# Step 6: Compare lines across books
python tools/compare_lines.py

# Step 7: Print ranked picks to terminal
python tools/generate_report.py

# Step 8: Push picks to Google Sheet
python tools/export_to_sheets.py
```

## Recommended Timing

- **Run at:** 2–3 hours before first puck drop (lines are usually posted by then)
- **Re-run `fetch_odds.py`** closer to game time if lines shifted or odds were missing
- **Do not run after puck drop** — no actionable use

---

## After the Games: Track Results

The next morning:
1. Open the Google Sheet → "Picks" tab
2. Fill in the **Result** column with each player's actual SOG
   - Box scores: [NHL.com](https://www.nhl.com/scores) or ESPN
3. Export labeled rows to a CSV for model training:
   ```
   player_key,game_date,actual_sog,line,book,over_odds
   ```
4. Append to your growing `historical_labels.csv`

After 50+ labeled games → run `train_model.py` to upgrade from Phase 1 baseline to Phase 2 logistic regression.

## Troubleshooting

| Problem | Likely cause | Fix |
|---------|-------------|-----|
| No players in features file | Schedule teams don't match player log team abbreviations | Check NHL API team abbrev format (e.g. "VAN" not "Vancouver") |
| `fetch_odds.py` returns no players | SOG market not yet posted for today's games | Re-run 1–2 hours later |
| Player missing from odds output | Name normalization mismatch | Check `fetch_odds.py::name_to_key()` vs NHL API name |
| Google Sheet auth error | Token expired | Delete `token.json`, re-run `export_to_sheets.py` |
| `model.pkl` not found | Model not yet trained | Normal — falls back to Phase 1 baseline automatically |
| Low confidence on all plays | Early season or noisy rolling averages | Expected — confidence improves as game logs grow |
